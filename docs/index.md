## Design Rules (for developer)

- Copy should read like a person, not a pitch deck. No signal sentences.
- One screen hero. White space > content density.
- No animated typing effects, snake GIFs, or matrix rain.
- Experience is compact — one line per role, no bullet points.
- Projects speak for themselves — short descriptions, links to code.
- Contact CTA prominent but not aggressive.
- No visitor counters, no resume-dump experience blocks.
- Stack is understated, not a hero element.
- Quarterly refresh: update status, swap featured projects.
